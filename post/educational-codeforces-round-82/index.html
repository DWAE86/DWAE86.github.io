B. National Project（2s 256Mb）

题目大意

有nnn块砖需要维修。如果在好天气里修变成好砖，在坏天气里变成坏砖。不一定每天都必须修砖。一开始先是连续ggg天好天气，又是连续bbb天坏天气，又是天好天气，循环往复，求至少需要多少天，使得nnn块砖全部被修，且至少有⌈n2⌉\left \lceil \frac{n}{2} \right \rceil⌈2n​⌉块砖是好砖。

分析

比赛时无限接近答案，不过还是没有太想清楚。题目要求我们满足两个条件，一个是nnn块砖全部被修，另一个是至少有⌈n2⌉\left \lceil \frac{n}{2} \right \rceil⌈2n​⌉块砖是好砖。第一个说明答案至少是nnn。第二个我们只需要求得得到⌈n2⌉\left \lceil \frac{n}{2} \right \rceil⌈2n​⌉好天气所需要的总天数，因为坏天气时可以选择不修。求法简单，用⌈n2⌉\left \lceil \frac{n}{2} \right \rceil⌈2n​⌉除以ggg得到ddd，如过有剩余则是d⋅(g+b)+(⌈n2⌉mod  g)d\cdot (g+b)+(\left \lceil \frac{n}{2} \right \rceil\mod g)d⋅(g+b)+(⌈2n​⌉modg)，否则是d⋅(g+b)−bd\cdot (g+b)-bd⋅(g+b)−b，因为最后连续bbb天不需要。答案就是与nnn第二种得到结果两个中更大的。

E. Erase Subsequences（2s 256Mb）

题目大意

给一个字符串sss，目标字符串ttt。一开始你手里有一个空串，你可以从sss中选择子序列，从sss中删去，将序列排在你手里的串右边（+=），最多这样做两次。问可否得到ttt。

∣s∣≤4⋅102|s| \leq 4\cdot 10^2∣s∣≤4⋅102

分析

dp题。第一步我也想到，枚举ttt的分割点，前后字串为aaa和bbb，关键是如何在前后两次拿取不影响的情况下dp。令数组dp[lena][lenb]dp[lena][lenb]dp[lena][lenb]表示拿到aaa的前lenalenalena个，bbb的lenblenblenb前个字符需要的最少的lenslenslens，我们只要预处理每个位置后每种字符第一次出现的位置即可O(1)O(1)O(1)向后更新。总复杂度O(n3)O(n^3)O(n3)。

代码

int work()
{
	memset(dp, 63, sizeof(dp));
	dp[0][0] = 0;
	for(int i = 0; i <= lena; i++)
	for(int j = 0; j <= lenb; j++) if(dp[i][j] < lens)
	{
        // 更新状态，nt为预处理的位置
		if(i < lena) dp[i+1][j] = min(dp[i+1][j], nt[t[i+1]][dp[i][j]+1]);
		if(j < lenb) dp[i][j+1] = min(dp[i][j+1], nt[t[lena+j+1]][dp[i][j]+1]);
	}
	return dp[lena][lenb] <= lens; // 判断能否完成
}
 
int main()
{
	int T = read(); while(T--)
	{
		cle();
		in(s, lens); in(t, lent);

		for(int i = 1; i <= 26; i++)
		for(int j = lens; j; j--) nt[i][j] = (s[j]==i) ? j : nt[i][j+1]; // 预处理

		int flag = 0;
		for(lena = 1; lena <= lent; lena++) // 枚举分割点
		{
			lenb = lent-lena;
			if(work())
			{
				printf("YES\n");
				flag = 1;
				break;
			}
		}
		if(!flag) printf("NO\n");
	}
	return 0;
}
